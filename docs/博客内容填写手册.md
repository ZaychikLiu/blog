# 博客内容填写手册

如何填写个人信息、写文章、管理标签、图片、评论、统计和推荐文章

适用目录：`/Users/zaychik/Desktop/博客/hexo-icarus-blog-template`

## 1. 本地预览

进入博客目录并启动预览：

```bash
cd /Users/zaychik/Desktop/博客/hexo-icarus-blog-template
npm install
npm run dev
```

浏览器打开：

```bash
http://localhost:4000/blog/
```

> 如果 4000 端口被占用，可以运行 npx hexo server --port 4001，然后打开 http://localhost:4001/blog/。

## 2. 修改站点基本信息

主要改两个文件：_config.yml 管站点和路径，_config.icarus.yml 管主题外观和侧栏。

```bash
title: Zaychik's Blog
description: Notes on research, engineering, and everyday learning.
author: Zaychik Liu
url: https://zaychikliu.github.io/blog
root: /blog/
```

主题标题和个人卡片在 _config.icarus.yml：

```bash
logo:
  text: "Zaychik's Blog"

widgets:
  - position: left
    type: profile
    author: Zaychik Liu
    avatar: /img/avatar.svg
    follow_link: https://github.com/ZaychikLiu
```

- 博客标题：改 title 和 logo.text。
- 作者名：改 author 和 profile author。
- 头像：替换 source/img/avatar.svg。
- 社交链接：改 social_links 里的 URL。

## 3. 创建新文章

推荐用 Hexo 命令创建文章：

```bash
npm run new "My First Post"
```

文章会出现在 source/_posts/。文章头部 front matter 示例：

```bash
---
title: My First Post
date: 2026-06-10 20:00:00
tags:
  - Python
  - Research
categories:
  - Notes
---

首页摘要写在这里。

<!-- more -->

正文从这里开始。
```

> <!-- more --> 前面的内容会显示在首页列表里；后面的内容只在点进文章后显示。

## 4. 标签、分类和归档

你不用手工维护 Archives 和 Tags 页面。只要在文章 front matter 里写 tags/categories，Hexo 会自动生成页面。

```bash
tags:
  - Deep Learning
  - PyTorch
categories:
  - Research
```

自动生成的地址：

- /blog/tags/
- /blog/archives/
- /blog/categories/

如果想在导航栏加入 Categories，改 _config.icarus.yml：

```bash
navbar:
  menu:
    Archives: /archives
    Categories: /categories
    Tags: /tags
    About Me: /about
```

## 5. About Me 页面

编辑这个文件：

```bash
source/about/index.md
```

建议包含：

- 一句话介绍你是谁。
- 研究方向或工程方向。
- 当前项目和代表作品。
- GitHub、Google Scholar、LinkedIn、邮箱。
- 你希望读者如何联系你。

## 6. 图片、附件和公式

模板启用了 post_asset_folder。文章 my-post.md 可以配同名资源目录：

```bash
source/_posts/my-post.md
source/_posts/my-post/figure1.png
```

文章里引用图片：

```bash
![figure](figure1.png)
```

数学公式写法：

```bash
Inline: $E = mc^2$

Block:

$$
\sum_i x_i
$$
```

## 7. Popular Posts 推荐文章

模板已经加入类似 ppwwyyxx 博客侧栏的 Popular Posts。编辑 _config.yml：

```bash
advanced:
  popular_posts:
    enabled: true
    title: Popular Posts
    posts:
      - title: Hello Hexo Icarus
        url: /2026/hello-hexo-icarus/
      - title: Research Note Template
        url: /2026/research-note-template/
```

每次想长期推荐一篇文章，就把标题和 URL 加到 posts 下面。

## 8. 文章目录、分享和阅读进度条

这些高级功能都在 _config.yml 的 advanced 区块里。

```bash
advanced:
  reading_progress: true
  heading_anchors: true
  share:
    enabled: true
    platforms:
      - x
      - linkedin
      - email
      - copy
  toc:
    enabled: true
    title: Contents
    min_headings: 1
```

- reading_progress：页面顶部蓝色阅读进度条。
- heading_anchors：标题旁边的 # 锚点。
- share：文章底部分享按钮。
- toc：文章内目录栏，默认只要有 h2/h3 标题就显示在侧栏。

## 9. 评论系统

模板默认启用 Utterances 评论区，读者可以用 GitHub 账号在文章底部留言。上线后需要安装 Utterances GitHub App，并确认仓库 Issues 已开启。

### Utterances

1. 安装 https://github.com/apps/utterances。
2. 允许它访问 ZaychikLiu/blog 仓库。
3. 打开仓库 Settings -> Features，确认 Issues 已开启。
4. 部署后点进任意文章底部，确认评论框出现。

```bash
advanced:
  comments:
    enabled: true
    provider: utterances
    utterances_repo: ZaychikLiu/blog
    utterances_issue_term: pathname
    utterances_theme: github-light
```

### Giscus

如果你更喜欢 GitHub Discussions 风格，也可以改用 Giscus；那时需要在仓库开启 Discussions，并到 https://giscus.app 生成 repo_id 和 category_id。

```bash
advanced:
  comments:
    enabled: true
    provider: giscus
    repo: ZaychikLiu/blog
    repo_id: 这里填 data-repo-id
    category: General
    category_id: 这里填 data-category-id
    mapping: pathname
    theme: light
    lang: en
```

## 10. 统计和 SEO

Google Analytics 4：创建 Web 数据流后，把 Measurement ID 填进去。

```bash
advanced:
  analytics:
    google_analytics:
      measurement_id: G-XXXXXXXXXX
```

Plausible：

```bash
advanced:
  analytics:
    plausible:
      domain: zaychikliu.github.io
      script_url: https://plausible.io/js/script.js
```

SEO 文件已经配置好，部署后检查：

```bash
https://zaychikliu.github.io/blog/sitemap.xml
https://zaychikliu.github.io/blog/robots.txt
```

## 11. 发布前检查清单

1. 运行 npm run build，确认无错误。
2. 本地打开 http://localhost:4000/blog/ 看首页。
3. 点进一篇文章，看分享按钮、目录、代码块、公式是否正常。
4. 运行 git status，确认只提交你想提交的文件。
5. git add . && git commit && git push。

```bash
npm run build
git status
git add .
git commit -m "Update blog content"
git push
```
